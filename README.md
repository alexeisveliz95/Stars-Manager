Welcome to the engine behind my content curation. **Stars-Manager** is an automated system built with Python and GitHub Actions that acts as an intelligence radar and personal knowledge manager.

## 🚀 What does this system do?
Instead of having a messy list of bookmarked repositories, this bot runs periodically to:
1. **Sync:** Fetches my starred repositories using the GitHub API.
2. **Classify (AI & Rules):** Evaluates names, descriptions, and *topics* to categorize each project (AI, Web3, Cybersecurity, PlayStation Linux, etc.).
3. **Generate Markdown:** Creates a modular note structure ready to be seamlessly integrated with **Obsidian** (PKM).

## 📊 Explore the Data
All the information classified and updated by the bot is located in the main index:

👉 **[GO TO THE REPOSITORIES DASHBOARD](DASHBOARD.md)** 👈

## 🛠️ Architecture
The project uses a modular approach to ensure scalability, separating the extraction logic (scrapers), the classification engine, and the Markdown file generation. Everything is orchestrated serverless through CI/CD pipelines (GitHub Actions).

# 🇪🇸 En Español

Bienvenido al motor detrás de mi curación de contenido. **Stars-Manager** es un sistema automatizado construido con Python y GitHub Actions que actúa como un radar de inteligencia y gestor de conocimientos.

## 🚀 ¿Qué hace este sistema?
En lugar de tener una lista desordenada de repositorios guardados, este bot se ejecuta periódicamente para:
1. **Sincronizar:** Extrae mis repositorios favoritos usando la API de GitHub.
2. **Clasificar (AI & Rules):** Evalúa nombres, descripciones y *topics* para categorizar cada proyecto (AI, Web3, Ciberseguridad, PlayStation Linux, etc.).
3. **Generar Markdown:** Crea una estructura de notas modulares lista para integrarse con **Obsidian** (Personal Knowledge Management).

## 📊 Explora los Datos
Toda la información clasificada y actualizada por el bot se encuentra en el índice principal:

👉 **[IR AL DASHBOARD DE REPOSITORIOS](DASHBOARD.md)** 👈

## 🛠️ Arquitectura
El proyecto utiliza un enfoque modular para asegurar su escalabilidad, separando la lógica de extracción (scrapers), el motor de clasificación y la generación de archivos Markdown. Todo orquestado sin servidores mediante flujos de CI/CD (GitHub Actions).
